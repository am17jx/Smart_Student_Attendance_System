# التقنيات والأدوات المستخدمة (Technologies Used)

يعتمد نظام الحضور الذكي والآمن (Secure Smart Attendance System) على مجموعة من التقنيات والمكتبات الحديثة لضمان الأمان، الأداء العالي، وتجربة المستخدم السلسة. فيما يلي تفصيل للتقنيات المستخدمة في بناء النظام:

## 1. تطوير الواجهة الأمامية (Frontend Development)
تُركز الواجهة الأمامية على تفاعل المستخدم وتقديم تجربة سريعة وسلسة:

*   **لغة البرمجة الأساسية:** TypeScript (نسخة مطورة من JavaScript تضمن كتابة كود آمن من الأخطاء).
*   **إطار العمل (Framework):** React.js (الإصدار 18).
*   **أداة بناء المشروع (Build Tool):** Vite (لبناء وتطوير سريع جداً للواجهة).
*   **توجيه الصفحات (Routing):** React Router DOM.
*   **إدارة الحالة والبيانات (State Management):** React Query / TanStack Query (لجلب البيانات وتخزينها مؤقتاً بكفاءة عالية).
*   **مكتبة واجهة المستخدم (UI Library):** Shadcn UI و Radix Primitives.
*   **تنسيق الواجهة (Styling):** Tailwind CSS (لتصميم متجاوب وسريع الاستجابة).
*   **الرسومات البيانية (Data Visualization):** Recharts.
*   **تحليل وقراءة QR Code:** مكتبة `html5-qrcode`.

## 2. تطوير الواجهة الخلفية (Backend Development)
تعمل الواجهة الخلفية كمحرك للنظام مبني للتعامل مع العمليات المعقدة وأمن البيانات:

*   **بيئة التشغيل الأساسية:** Node.js.
*   **إطار العمل (Framework):** Express.js (لبناء RESTful APIs).
*   **لغة البرمجة:** TypeScript.
*   **قاعدة البيانات (Database):** PostgreSQL (قاعدة بيانات علائقية قوية وموثوقة).
*   **مُعرّف قواعد البيانات (ORM):** Prisma (للتعامل الآمن مع قاعدة البيانات).
*   **التخزين المؤقت (Caching):** Redis (عبر `ioredis` لحفظ الجلسات المؤقتة وحالات الـ QR).
*   **المعالجة الجغرافية (Geolocation):** `@turf/turf` و `geolib` لحساب المسافات والتأكد من تواجد الطالب ضمن النطاق الجغرافي (Geo-fencing).
*   **توليد التقارير (Report Generation):** Puppeteer و PDFKit لإنشاء وتصدير تقارير الحضور كملفات PDF.
*   **إرسال البريد الإلكتروني (Email Service):** Nodemailer (لإرسال الإنذارات وإعادة تعيين كلمات المرور).

## 3. تقنيات الأمان والخصوصية (Security & Privacy Technologies)
هذه التقنيات تميز المشروع عن الأنظمة التقليدية وتضيف طبقات حماية متعددة:

*   **المصادقة (Authentication):** JSON Web Tokens (JWT) لإدارة الجلسات بشكل آمن.
*   **التشفير (Encryption):** `bcryptjs` لتشفير كلمات المرور في قاعدة البيانات.
*   **البصمة الرقمية للجهاز (Device Fingerprinting):** `@fingerprintjs/fingerprintjs` (في الواجهة الأمامية) لمنع استخدام نفس الجهاز لتسجيل حضور أكثر من طالب.
*   **تشفير ورموز الاستجابة السريعة (QR Security):** تقنية Dynamic QR (مكتبة `qrcode` مع نظام تبديل مستمر للرمز لمنع تصويره ومشاركته).
*   **حماية الواجهة البرمجية API:**
    *   `helmet`: لحماية عناوين HTTP (HTTP Headers).
    *   `express-rate-limit`: لمنع هجمات القوة الغاشمة (Brute Force) والضغط على السيرفر.
    *   `cors`: للتحكم في الوصول المشترك (Cross-Origin Resource Sharing).
*   **تعتيم الشفرة (Code Obfuscation):** `javascript-obfuscator` لتعقيد شيفرة الواجهة الأمامية عند النشر لمنع الهندسة العكسية.

## 4. البنية التحتية والتشغيل الموحد (Infrastructure & DevOps)
تم استخدام هذه الأدوات لضمان بيئة تشغيل موحدة وسهلة النشر:

*   **تشغيل الحاويات (Containerization):** Docker (لتغليف كل جزء من التطبيق في حاوية مستقلة).
*   **إدارة الحاويات المتعددة (Orchestration):** Docker Compose (لربط Backend, Frontend, Traefik, PostgreSQL معاً).
*   **موجه الحافة والخادم العكسي (Edge Router / Reverse Proxy):** Traefik (لإدارة توجيه الطلبات ديناميكياً) بالإضافة لـ Nginx (كخادم للملفات الثابتة للواجهة الأمامية).

*(ملاحظة بخصوص الذكاء الاصطناعي: النظام حالياً لا يعتمد على نماذج تعلم آلي عميقة (Machine Learning) بحد ذاته، بل يعتمد على خوارزميات تحقق دقيقة من البصمة الرقمية والموقع الجغرافي وديناميكية الرموز لتحقيق نسبة عالية جداً من الدقة والأمان في تسجيل الحضور).*
