# دليل البداية لرفع مشروعك على السيرفر (Deployment Guide) 🚀

هذا الدليل أُعد خصيصاً ليكون مرجعك الشامل لتتعلم كيف تأخذ مشروعاً يعمل محلياً (باستخدام Docker) وترفعه ليعمل على سيرفر حقيقي على الإنترنت. مشروعك (Attendance System) مصمم بطريقة ممتازة وموزع على حاويات (Containers)، مما يجعل عملية رفعه احترافية وسلسة.

---

## 🏗️ 1. المفاهيم الأساسية (ماذا يحدث خلف الكواليس؟)
قبل أن نبدأ بالتطبيق، من المهم أن تفهم هذه المصطلحات لتطوير نفسك:
- **السيرفر السحابي (VPS):** هو جهاز كمبيوتر متصل بالإنترنت يعمل على مدار 24 ساعة (مثل DigitalOcean, AWS EC2, Hetzner, Linux).
- **Docker Compose:** أداة تُشغل مشروعك بكل أجزائه (واجهة، باك إند، قاعدة بيانات) بضغطة زر واحدة أينما كان السيرفر.
- **Traefik/Nginx (Reverse Proxy):** حارس البوابة؛ يستقبل الطلبات من المستخدمين (مثل `api.attend.com`) ويوجهها للحاوية الصحيحة.
- **Domain Name:** النطاق (مثل `yoursite.com`)، يتم ربطه برقم الآي بي (IP) الخاص بسيرفرك ليتمكن الناس من تصفح المشروع.

---

## 🛠️ 2. المتطلبات الأولية للرفع
لتتمكن من رفع المشروع، ستحتاج إلى:
1. **استئجار سيرفر يعتمد على Linux (يُفضل Ubuntu 22.04 LTS أو 24.04).**
   - *نصيحة:* للمشاريع المبدئية، سيرفر بمواصفات "2GB RAM و 2 CPU" بسعر حوالي 5-10 دولار شهرياً سيكون ممتازاً للتعلم.
2. **شراء نطاق (Domain Name) أو تجربة مجانية (اختياري لكن مفضل للإنتاج).**
3. **الوصول للسيرفر عبر أداة SSH.**

---

## 🧑‍💻 3. خطوات الرفع (الجانب العملي)

### الخطوة الأولى: الدخول إلى السيرفر وتجهيزه
بمجرد شرائك سيرفر، ستحصل على (IP Address) وكلمة مرور أو مفتاح.
من جهازك (Windows Terminal)، قم بالاتصال بالسيرفر:
```bash
ssh root@YOUR_SERVER_IP
```
بعد الدخول، قم بتحديث النظام:
```bash
sudo apt update && sudo apt upgrade -y
```

### الخطوة الثانية: تثبيت Docker و Docker Compose
بما أن مشروعك مجهز بـ `docker-compose.yml`، فلن نحتاج لتثبيت Node.js أو قاعدة البيانات يدوياً!
قم بتثبيت Docker عبر الأمر الرسمي السريع:
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
```

### الخطوة الثالثة: نقل كود المشروع إلى السيرفر
أفضل وأجمل طريقة لنقل مشروعك هي باستخدام **Git (GitHub/GitLab)**:
1. ارفع الكود الخاص بك من جهازك (Windows) إلى مستودع خاص (Private Repo) على GitHub.
2. في السيرفر (Linux)، قم بعمل استنساخ للمستودع:
```bash
git clone https://github.com/YourName/system-attendnce.git
cd system-attendnce
```

### الخطوة الرابعة: ضبط ملفات البيئة `.env`
لا تقم أبداً برفع ملف `.env` الذي يحتوي على أسرار (Passwords) لموقع GitHub.
بدلاً من ذلك، في السيرفر، اصنع ملف `.env` جديد:
```bash
cp Back-end/.env.example Back-end/.env
nano Back-end/.env
```
*(عدّل المتغيرات لتناسب الإنتاج (Production)، مثل تغيير الـ `JWT_SECRET` وتعديل `FRONTEND_URL` ليكون النطاق الحقيقي للشرِكة أو السيرفر، ولا تنسَ استخدام كلمة مرور قوية لقاعدة البيانات!)*

### الخطوة الخامسة: التشغيل والبناء!
أنت الآن جاهز. بنفس الأمر الذي استخدمته في الويندوز:
```bash
docker compose up -d --build
```
سيبدأ السيرفر بتحميل وبناء أجزاء المشروع، وبعد بضع دقائق سيكون مشروعك يعمل في الخلفية على السيرفر!

---

## 🌐 4. كيف سيتعامل الناس مع مشروعك؟ (الدومين و Traefik)
مشروعك يحتوي أصلاً في `docker-compose.yml` على أداة Traefik!
لكي يقرأ Traefik النطاق الخاص بك، ويقوم بتأمين الاتصال (إضافة قفل SSL / HTTPS) أوتوماتيكياً:
1. اذهب لمزود الدومين الخاص بك (مثل Namecheap أو Cloudflare).
2. قم بإضافة سجل (A Record) يربط النطاق `yoursite.com` بالآي بي الخاص بالسيرفر.
3. عدّل إعدادات `docker-compose.yml` وأضف القواعد الخاصة بالنطاق في قسم الـ `labels:` أسفل خدمة `frontend` و `backend`.
*(يُفضل أن تدرس توثيق Traefik لتعلم كيفية تفعيل Let's Encrypt SSL).*

---

## 🛡️ 5. ممارسات أمنية (Best Practices)
بما أنك أصبحت مدير سيرفر، هذه أهم القواعد لتطور نفسك كمهندس:
1. **شغّل الجدار الناري (UFW):** اسمح فقط بمرور الترافيك عبر البورتات 80 (HTTP) و 443 (HTTPS) و 22 (SSH).
2. **لا تستخدم حساب Root للعمل اليومي:** قم بإنشاء مستخدم عادي وأعطه صلاحية `sudo`.
3. **تغيير منفذ الـ SSH:** بدلاً من البورت 22، غيّره لرقم آخر لتفادي هجمات التخمين (Brute-force).
4. **نسخ احتياطية (Backups):** اصنع سيناريو مصغر لنقل بيانات قاعدة البيانات (PostgreSQL volume) إلى مكان آخر يومياً كي لا تفقد بيانات النظام أبداً.
